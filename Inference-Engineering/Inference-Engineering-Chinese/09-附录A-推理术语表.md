# Appendix A: Inference Glossary（推理术语表）

Activation function - 激活函数：一种（通常是）可微的非线性函数，如 ReLU，插入在线性层之间，以防止多层神经网络退化为单次 matmul（矩阵乘法）。

Active–active - 双活模式：一种高可用性部署模式，多个区域/集群同时主动服务线上流量。如果任何一个平面发生故障，流量会无缝地继续在其他平面上运行。

Active–passive - 主备模式：一种故障转移模式，保持一个"热备"集群或区域处于就绪但空闲状态。如果主动平面发生故障，流量将切换到备用平面。

Ada Lovelace (architecture) - Ada Lovelace 架构：NVIDIA 面向图形的 GPU 架构，与 Hopper 一起于 2022 年发布。适用于小型模型和对成本敏感的工作负载，不适合大规模 LLM 推理。

Agent - 智能体：一种不仅仅提供信息，还能执行操作的 AI 应用。Agent 工作流通常依赖多次推理调用，通常跨多个模型和模态，并且需要访问工具。

AI‑native application - AI 原生应用：核心用户体验和价值依赖于生成模型的产品。推理选择是应用约束的下游决策：模态、延迟预算、单位经济效益和使用模式。

Ampere (architecture) - Ampere 架构：一种较旧的 NVIDIA GPU 架构，仍在传统或小规模部署中使用。Hopper 和 Blackwell 架构在原始速度和规模化成本方面通常都优于 Ampere。

Application Programming Interface (API) - 应用程序编程接口：用于发送请求和接收响应的结构化接口。推理引擎暴露一个 API 用于向模型发起查询。

Arithmetic intensity - 算术强度：给定算法中每移动一个字节所执行的操作数。与 GPU 的 ops:byte 比值相比，算术强度指示 kernel 是受计算约束还是受内存约束。

Attention - 注意力：transformer 的核心机制，通过 Q/K/V 投影和 softmax 将一个 token 与先前的 token 关联起来。由于注意力的计算和内存需求很高，它是优化的主要目标。

Automatic Speech Recognition (ASR) - 自动语音识别：音频输入、文本输出的转录模型（如 Whisper）。Decoder 工作主导运行时间，受益于 LLM 风格的优化和 in‑flight batching。

Autoregressive token generation - 自回归 token 生成：每个 token 依赖于之前每个 token 的迭代生成方式。自回归 token 生成分为两个阶段：处理输入 token 的 prefill 阶段，以及生成输出 token 的 decode 阶段。

Autoscaling - 自动扩缩容：根据流量或利用率自动增减服务某个模型的副本数量。自动扩缩容使容量与需求匹配，维持延迟 SLA 并最小化浪费的支出。

Autoscaling window - 扩缩容窗口：用于决定扩容/缩容操作的滚动时间窗口。较长的窗口保持副本数量稳定；较短的窗口对流量尖峰反应更快。

B200：NVIDIA 基于 Blackwell 的数据中心 GPU，具有 192 GB VRAM、8 TB/s 内存带宽和 5 petaFLOPS 的 FP8 算力。

B300：NVIDIA 基于 Blackwell 的数据中心 GPU，具有 288 GB VRAM、8 TB/s 内存带宽和 5 petaFLOPS 的 FP8 算力。

Bandwidth - 带宽：每秒可以通过 VRAM 等内存或 NVLink 等互连传输的数据量。

Baselines - 基线：在应用优化之前，仔细记录的性能和质量测量值。基线使增益或退化的归因变得清晰。

Basic Linear Algebra Subprograms (BLAS) - 基本线性代数子程序：线性代数基本操作的标准接口。

Batch - 批处理：同时处理多个输入，在 LLM 推理中很常见。

Batch sizing - 批大小设置：推理引擎的核心延迟-吞吐量杠杆。更大的批提高总吞吐量但恶化每个用户的延迟。

Benchmark (intelligence) - 基准测试（智能）：衡量模型正确回答问题或采取适当行动能力的测试（如 MMLU）。

Benchmark (performance) - 基准测试（性能）：在给定模型和定义的工作负载下，对推理服务延迟和吞吐量的测量。

BF16：一种 16 位浮点格式，指数位比 FP16 更大，适用于训练，有时也用于推理。更高的动态范围有助于保留离群值。

Bin packing (multi‑cloud) - 装箱策略（多云）：将跨云、区域和集群的异构 GPU 池视为单一可调度资源的实践，由多云容量管理基础设施支持。

Blackwell (architecture) - Blackwell 架构：NVIDIA 于 2024 年末推出的 GPU 代际，支持 FP4、microscaling 格式（MXFP8、MXFP4、NVFP4）和高内存带宽。

Blue‑green deployment - 蓝绿部署：两个并行的生产环境（"蓝"和"绿"）；在它们之间切换流量以实现零停机部署和快速回滚。

Cache‑aware routing - 缓存感知路由：将请求导向已经持有匹配前缀或所需 LoRA 的副本。更高的缓存命中率带来更低的 TTFT。

Canary deployment - 金丝雀部署：一种新的生产部署，最初将少量线上流量导向新部署以验证稳定性和性能。随着时间推移，新部署逐步接管所有生产流量。

Causal language model (CLM) - 因果语言模型：一种 decoder‑only transformer，在给定先前上下文的情况下预测下一个 token。本书中的所有生成式 LLM 都是 CLM。

Central Processing Unit (CPU) - 中央处理器：一种针对顺序工作负载优化的通用处理器。CPU 用于编排、调度、网络和预处理，但很少直接处理生成式 AI 推理。

Chat template - 聊天模板：消息的模型特定格式化和序列化（角色、分隔符、序列开始/结束 token）。

Chunked prefill - 分块 prefill：将长输入分成多个块，并将 prefill 与 decode 或其他工作重叠，防止单个长序列独占资源。

Classifier-free guidance - 无分类器引导：在图像生成的每一步中平衡无条件去噪和提示条件去噪。较低的引导增强创意性；较高的引导强化提示遵从性。

CLIP (text encoder) - CLIP（文本编码器）：用于早期图像流水线（如 SDXL）的文本/图像编码器。现代系统通常替换为完整的 LLM 以获得更强的提示理解能力。

Closed model - 闭源模型：权重不可用的专有模型，如 GPT-5、Claude Sonnet 或 Google Gemini。

Cold start - 冷启动：将副本从零扩展到第一个成功响应所需的时间（步骤包括 GPU 供应、容器启动、模型加载、推理引擎编译）。

ComfyUI：一种用于组装图像流水线（基础模型、refiner、LoRA、ControlNet）的工作流工具。鼓励模块化、可替换的组件。

Compute‑bound - 计算受限：一种受可用 FLOPS 而非内存带宽限制的算法。LLM prefill 和图像/视频生成通常是计算受限的。

Context Parallelism (CP) - 上下文并行：跨 GPU 复制权重并划分注意力上下文。对于注意力在巨大 latent space 上工作的视频模型至关重要。

Context window - 上下文窗口：模型在单个请求中可以跨输入、推理和输出处理的最大 token 数量。

Continuous batching (in‑flight) - 连续批处理（in‑flight）：请求的 token 级别交错，使 GPU 插槽始终被利用。最小化批处理带来的每个用户的延迟损失。

Control plane (multi‑cloud) - 控制平面（多云）：用于部署模型和分配资源的全局编排器。

Core (CUDA) - CUDA Core：一种通用算术单元，执行广泛的标量和逐元素操作。

Core (Tensor) - Tensor Core：一种专用硬件单元，针对混合精度矩阵乘累加（MMA）操作进行优化。Tensor Core 是推理中最重要的计算类型。

Cross‑attention - 交叉注意力：将一个序列（Q）条件化到另一个序列的 K/V 上（例如，文本条件化图像）。常见于多模态和去噪流水线。

cuBLAS：CUDA 的 BLAS 实现，提供高质量的 GEMM 和相关原语。

CUDA：NVIDIA 的编程模型和平台，用于 GPU kernel、graph、内存和执行。

CUDA driver - CUDA 驱动：应用程序和 GPU 硬件之间的低层接口，用于管理内存和执行。

CUDA graph - CUDA 图：由 kernel 和其他 GPU 操作组成的有向无环图（DAG），用于优化重复工作流。

CUDA kernel - CUDA 核函数：一种用户定义的函数，在 GPU 上执行并行化代码。

CUDA runtime - CUDA 运行时：一种面向开发者的 API，用于启动 kernel 和管理内存。

cuDNN：在 CUDA 中构建深度神经网络的原语。

CuTe：一种领域特定的 C++ 模板库，抽象分块张量操作，帮助开发者编写精度感知、硬件优化的 GEMM 和融合 kernel。

CUTLASS：一种 CUDA C++ 模板库，提供用于编写高性能、架构调优的 GEMM 和相关 kernel 的构建块。

Data sovereignty - 数据主权：关于模型输入和输出在地理上何处被处理和存储的法律约束。

Decode - 解码：LLM 推理中受内存约束的阶段，自回归生成循环每次前向传播产生一个 token。

DeepGEMM：由 DeepSeek AI 团队创建的干净高效的 GEMM kernel 库，在 FP8 中具有出色的性能。

Denoising model - 去噪模型：diffusion 流水线的核心，迭代地将 latent noise 细化为图像或视频。

Diarization - 说话人分离：按说话人对音频进行分段（"谁在何时说话"）；通常与 ASR 流水线中的 VAD 配合使用。

Diffusers (library) - Diffusers 库：图像和视频生成流水线的参考实现。

Disaggregation - 分离：将 prefill 和 decode 分离到独立扩展的引擎上，运行在独立的硬件资源上。

Distillation - 蒸馏：训练一个较小的学生模型来模拟较大的教师模型的概率分布（而不仅仅是输出），以在更少参数下保留模型行为。

Docker：容器化技术，用于构建包含推理服务及其依赖的标准化包。

Dockerfile：一种人类可读的文件，包含规范的、机器可解释的指令，用于创建镜像。

Dynamic batching - 动态批处理：动态批处理在批次满或短计时器到期时（以先到者为准）启动一个批次。平衡延迟稳定性与利用率；在 LLM 中已被 continuous batching 取代。

Dynamic range (quantization) - 动态范围（量化）：在数字格式中可以表示的绝对值范围。浮点数由于指数-尾数结构，在相同字节数下比整数具有更高的动态范围。

EAGLE (speculation) - EAGLE 推测：一种小型的、专门构建的 draft model，被训练来消费隐藏状态并提出多个 token，以在 speculative decoding 中获得高接受率。

Elo (quality meta‑metric) - Elo（质量元指标）：采用头对头胜率风格评分来比较模型质量。在智能基准测试之外提供有用的方向性信号。

Embedding model - 嵌入模型：将文本或图像输入编码为固定维度向量以进行语义相似性搜索，用于 RAG 和 agent 记忆。现代变体通常使用 LLM 骨干和 Matryoshka 表示。

Encoder - 编码器：将原始输入转换为内部表示的网络（如 Whisper 中的音频特征）。在 encoder‑decoder 模型中与 decoder 配对使用。

Evals - 评估：针对模型的、反映真实世界用例的任务特定测试，用于产品特定的模型智能测试。

Expert Parallelism (EP) - 专家并行：将 MoE 的专家分片到多个 GPU 上；每个 GPU 包含多个完整专家。以较低的 GPU 间通信开销增加总吞吐量。

Few‑step image generation - 少步图像生成：在八步或更少步骤内生成可用图像的模型。速度提升 80% 到 90%，但有明显的质量权衡；非常适合实时应用。

Feynman (architecture) - Feynman 架构：Rubin 之后的下一代 NVIDIA GPU 架构。细节有限；预期将继续强调低精度和内存带宽。

Fine‑tuning - 微调：将预训练的基础模型适配到特定领域，通常能使更小的模型满足质量需求。

FlashAttention：一系列优化的注意力 kernel，最小化内存流量。FlashAttention 3 为 Hopper 编写，FlashAttention 4 目标是 Blackwell。

Floating-point data formats - 浮点数据格式：推理中使用的精度，如 FP16、FP8 和 FP4，具有高动态范围和指数-尾数结构。

FLOPS：每秒浮点运算次数，通常在 Tensor Core 上测量。

Foundation model - 基础模型：在广泛数据上训练的模型，作为多种下游任务的基础。基础模型（如 GPT、Claude、Llama）通常通过微调或直接通过提示使用。

Function calling - 函数调用：也称为 tool calling 或 tool use，模型在给定一组可用函数和提示的情况下，返回包含所选函数及其参数的结构化输出。

GB200：一种 NVIDIA 超级芯片，通过高带宽 NVLink 芯片间连接将 Grace CPU 与 B200 GPU 配对。GB200 用于机架级 NVLink 系统如 NVL72，适用于 KV cache 卸载、LoRA 交换和其他受益于 NVLink-C2C 的技术。

General matrix-matrix multiplication (GEMM) - 通用矩阵-矩阵乘法：BLAS 中的一种算法，也是推理的关键操作。

Generative AI - 生成式 AI：一类模型，与预测性 ML 模型不同，通过学习训练数据的潜在模式，跨模态（文本、图像、音频、视频、代码）创建新内容。

Generative Pretrained Transformer (GPT) - 生成式预训练 Transformer：由 OpenAI 创建的一系列用于文本生成的大型语言模型。

GH200：一种 NVIDIA 超级芯片，通过高带宽 NVLink 芯片间连接将 Grace CPU 与 H200 GPU 配对。GH200 用于机架级 NVLink 系统如 NVL72，适用于 KV cache 卸载、LoRA 交换和其他受益于 NVLink-C2C 的技术。

Goodhart's Law - 古德哈特定律："当一个度量成为目标时，它就不再是一个好的度量。"

GPU node - GPU 节点：标准的 8 个互连 GPU 机箱，配备 NVLink 和 NVSwitch。

Grace CPU：基于 ARM 的 NVIDIA CPU，具有 CPU 和 GPU 之间的高带宽芯片间互连。与 Hopper 和 Blackwell GPU 配合使用。

Graphics Processing Unit (GPU) - 图形处理器：一种高度并行的处理器，最初设计用于图形渲染，现在广泛用于生成式 AI 模型的训练和推理。

gRPC：结构化的、模式优先的双向流式协议。

Head (attention) - 头（注意力）：层内的一次独立注意力计算。

High-Bandwidth Memory (HBM) - 高带宽内存：数据中心 GPU 上用于 VRAM 的内存。最新代际包括 HBM3、HBM3e 和 HBM4。

Hopper (architecture) - Hopper 架构：NVIDIA 2022 年的 GPU 代际，支持 FP8 和异步编程特性。

Hyperscaler - 超大规模云服务商：如 AWS 和 GCP 等通用云服务提供商。

Image generation pipeline - 图像生成流水线：图像生成的基础模型是由多个模型组成的流水线：文本编码器、迭代去噪器和 VAE。

Inference - 推理：在生产环境中提供 AI 模型服务。

Inference engine - 推理引擎：一种高性能运行时（如 vLLM、SGLang、TensorRT‑LLM），支持批处理、缓存、量化和推测等优化技术。

InfiniBand：用于跨多个节点扩展推理和训练的节点间互连。虽然 InfiniBand 带宽高于 Ethernet 等替代方案，但远低于 NVLink。

Input sequence - 输入序列：作为请求的一部分提供给模型的 token，在推理的 prefill 阶段处理。

Input Sequence Length (ISL) - 输入序列长度：给定请求的输入序列中的 token 数量。

Instance (cloud) - 实例（云）：供应的虚拟机，包括 GPU、CPU 和 RAM 资源、存储、网络和互连。

Integer data formats - 整数数据格式：如 INT8 和 INT4 等动态范围有限的数字格式。

Inter‑token latency (ITL) - token 间延迟：decode 阶段生成 token 之间的时间。可转换为感知的 TPS（例如，2 毫秒 ITL 相当于 500 TPS）。

In‑flight batching - in‑flight 批处理：参见 continuous batching。Token 级别交错，以实现高利用率和稳定延迟。

Iterative denoising (diffusion) - 迭代去噪（diffusion）：从噪声开始，逐步在 latent space 中细化为图像或视频。

Jitter traffic (bench) - 抖动流量（基准测试）：为请求到达时间和序列形状添加随机性，以比均匀或突发合成负载更接近地模拟真实流量。

Kernel fusion - Kernel 融合：将两个或多个 kernel 重新实现为一个处理两个操作的单一 kernel，避免不必要的内存往返。

KV cache - KV 缓存：存储每个 token 的 K/V 张量以避免重新计算注意力，将注意力等式从二次时间变为线性时间操作。

L0/L1/L2 caches (GPU) - L0/L1/L2 缓存（GPU）：用于指令、共享内存和全局缓存的片上缓存内存层次结构。

Large Language Model (LLM) - 大语言模型：一种接受文本提示并返回新文本序列的生成式 AI 模型。许多著名的生成式 AI 模型系列，包括 GPT、Claude、Llama 和 DeepSeek，都是 LLM。

Latency percentiles - 延迟百分位数：以百分位数为基础（P50/P90/P95/P99）测量延迟，以同时关注平均情况和最差用户体验。

Latent consistency - 潜在一致性：一种少步策略，直接预测目标 latent，可能会重复以进行细化。速度非常快；但保真度低于完整 diffusion。

Latent space (images/videos) - 潜在空间（图像/视频）：进行去噪的低维表示（如 128×128）。

LLM：Large language model，大语言模型（如 GPT-5、Llama、DeepSeek）。

Load testing - 负载测试：发送持续高流量以探测吞吐量限制、队列行为和自动扩缩容表现。

Local (edge) inference - 本地（边缘）推理：在手机和电脑等终端用户设备上运行推理。

Logit biasing - Logit 偏置：微调或约束 token 概率以引导结构化输出（如 JSON/tool call）。在采样前、logits 之后应用。

Logits：一个非归一化概率向量，模型词汇表中每个 token 对应一个，在 decode 期间每次前向传播生成。

Lookahead decoding - 前瞻解码：在推理期间构建 n‑gram，使得无需单独的模型即可进行 draft token 预测。

LoRA：Low-rank adaptation，低秩适配，一种轻量级微调方法，对模型产生小的改变。推理服务通常需要在单个基础模型之间切换数千个 LoRA。

Machine learning (ML) - 机器学习：用于分类和趋势预测等任务的预测性建模，与创建新颖输出的生成式 AI 相对。

Matmul：Matrix multiplication，矩阵乘法。

Matryoshka representations (embeddings) - Matryoshka 表示（嵌入）：嵌套向量方案，允许可变维度，向量的前部编码更多语义含义。允许在向量大小和嵌入质量之间进行权衡。

Medusa (speculation) - Medusa 推测：通过微调添加额外的 decoder head，以在每次前向传播中生成多个 draft token。

Microscaling formats - 微缩放格式：如 MXFP8、MXFP4 和 NVFP4 等浮点数据格式，使用带有小块尺度因子（如每 32 个元素）的分块量化来提高精度。

Mixture of Experts (MoE) - 混合专家：一种模型架构，其中线性层的权重被分离为稀疏专家。路由器在每次前向传播中激活一部分专家。

Model parallelism (overview) - 模型并行概述：通过 Tensor、Expert 或 Pipeline Parallelism 跨 GPU 分拆工作。并行策略取决于模型大小、拓扑结构和延迟与吞吐量目标。

Multi‑cloud capacity management - 多云容量管理：跨提供商和区域放置工作负载的全局调度器。

Multi‑Instance GPU (MIG) - 多实例 GPU：较大 Ampere、Hopper、Blackwell 和 Rubin GPU 中的功能，可以将 GPU 划分为多达 8 个内存切片和 7 个计算切片。

Multi‑node inference - 多节点推理：当一个 8 GPU 节点的 VRAM 不足以容纳权重、激活和 KV cache 时，使用 InfiniBand 跨两个或更多节点扩展。需要适当的并行策略，节点间使用 PP 或 EP，因为 TP 需要过多的 all‑to‑all 通信，不适合 InfiniBand。

Neocloud - 新兴云服务商：专注于 GPU 的专业云服务提供商，如 Coreweave 和 Nebius。

Neural audio codec - 神经音频编解码器：一种学习型编码器，将音频压缩为 token，以及配对的解码器，将 token 转换回音频。

NIM：由 NVIDIA 创建的针对特定模型的预打包容器化微服务。

Node - 节点：配备 NVLink/NVSwitch 的物理 8‑GPU 基本单元。多节点通过节点间的 InfiniBand 连接。

NVFP4：NVIDIA 的 4 位浮点 microscaling 数字格式，具有双尺度因子和块大小为 16 的分块量化。

NVIDIA Dynamo：一个开源分布式服务平台，用于 KV 重用、分离和多 GPU/多节点编排。

NVL72：一种机架级 Blackwell 系统，互连 72 个 GPU 和 36 个 CPU。专为以极高吞吐量服务超大型模型而设计。

NVLink：GPU 之间的一对一通信层，在 Blackwell 上高达 1800 GB/s，在 Hopper 上高达 900 GB/s。

NVSwitch：建立在 NVLink 之上的全互联通信层，用于节点内所有 GPU 之间的协调。

N‑gram speculation - N‑gram 推测：使用 prefill 中观察到的 n‑gram 在 decode 期间提出长 draft 序列。对代码补全极其有效。

Offline inference - 离线推理：大型作业的异步批处理，针对吞吐量和成本而非每个请求的延迟进行优化。

Omni‑modal - 全模态：接受多种输入模态（文本、图像、视频、音频）并产生多种输出模态的模型。

Online inference - 在线推理：实时请求服务，针对严格的延迟预算进行优化。

ONNX：一种模型的中间表示和运行时。

Open model - 开源模型：权重可自由获取的模型，如 Llama、DeepSeek 或 Whisper。

Ops:byte ratio (GPU) - Ops:byte 比值（GPU）：GPU 在给定精度下每字节内存带宽的峰值操作数。与算术强度比较以诊断瓶颈。

Output sequence - 输出序列：模型在推理的 decode 阶段生成的 token。

Output Sequence Length (OSL) - 输出序列长度：模型为给定请求生成的输出序列中的 token 数量。

Out‑of‑memory error (OOM) - 内存不足错误：GPU VRAM 不足以加载权重或执行推理时的常见故障。

PagedAttention：一种注意力优化，其中 KV 块存储在固定大小的页中以提高性能，特别是在长上下文情况下。

PCIe (GPU form factor) - PCIe（GPU 外形规格）：一种数据中心 GPU 的外形规格，使用标准 PCI Express 插槽连接。PCIe GPU 通常具有比同型号 SXM 变体更低的基础规格和更少的互连选项。

Perceived TPS - 感知 TPS：在流式输出期间单个用户观察到的每秒 token 数。这个延迟指标是人们通常说 TPS 时更具体的术语。

Pipeline Parallelism (PP) - 流水线并行：将层分成多个阶段跨 GPU 分布。虽然对于稠密模型的多节点部署可接受，但 PP 会在流水线中引入气泡，某些 GPU 在等待其他步骤完成时空闲。

Prefill - 预填充：LLM 推理中受计算约束的阶段，处理输入序列并构建 KV cache。

Prefix caching - 前缀缓存：跨请求重用共享前缀的 KV 以跳过 prefill。大幅改善代码补全、多轮对话和 agent 的 TTFT。

Pretraining - 预训练：在广泛语料库上进行的大规模（通常是自监督）训练，以创建基础模型。

Prompt - 提示：给模型的指令；对于 diffusion，还包括 negative prompt 和 step/guidance 参数。

PyTorch compile (torch.compile) - PyTorch 编译：针对特定 GPU 的图捕获和 kernel 选择/融合。缓存编译后的引擎以缩短冷启动时间。

PyTorch Profiler - PyTorch 性能分析器：一种开发者工具，测量每个操作的 CPU 和 GPU 时间及内存。

Quantization (post‑training) - 量化（训练后）：降低权重、激活以及可能的 KV cache 的精度，以减少计算和内存带宽需求。

Quantization‑aware training - 量化感知训练：一种训练技术，其中量化尺度被计算、权重被联合优化，使得最终模型已经为低精度部署做好了校准。

Queue (request) - 队列（请求）：在自动扩缩容启动副本时暂存多余流量。

Real‑time factor (RTF) - 实时因子：衡量 ASR 模型转录音频速度的指标。在 6 秒内转录一小时音频的 RTF 为 600。

Retrieval‑augmented generation (RAG) - 检索增强生成：一种常见的应用模式，为 LLM 获取提示之外的额外上下文。

Ring attention - 环形注意力：一种 Context Parallelism 机制，GPU 以环形方式传递部分注意力结果。减少超大上下文的 all‑to‑all 压力。

Roofline model - 屋顶线模型：绘制算术强度与带宽和计算上限的图表，创建关于优化内存还是计算的视觉指南。

Rotary positional embeddings (RoPE) - 旋转位置嵌入：一种位置编码方案，将位置编码为学习的旋转，改善长上下文外推，代价是推理期间注意力需要更高的内存需求。

Routing (inference) - 路由（推理）：基于负载、KV cache、可用 LoRA 和序列形状将请求放置到副本上，以提高速度和利用率。

Rubin (architecture) - Rubin 架构：下一代 NVIDIA GPU 代际（2026 年），引入 HBM4 和 CPX 以应对计算受限的工作负载。

Sampling (decode) - 采样（解码）：根据生成的 logits 选择输出 token 的过程。常见策略包括贪心（argmax）、基于温度的采样、top‑k 和 top‑p（nucleus）采样。

Scale factor (quantization) - 缩放因子（量化）：用于将低精度值映射回其原始数字格式的乘数。

Scale to zero - 缩容至零：空闲时关闭所有副本；按需启动。需要快速冷启动和健壮的队列；最适合可预测或开发工作负载。

SDXL：一种具有指导意义的早期 diffusion 图像流水线（base + refiner + CLIP）。现代系统保留了这种结构，但使用更大、更强大的组件。

Service Level Agreement (SLA) - 服务等级协议：对系统延迟、吞吐量、正常运行时间或其他性能因素的合同承诺。

Service Level Objective (SLO) - 服务等级目标：旨在满足或超过给定系统 SLA 的内部目标。

SGLang：一种快速推理引擎，具有灵活的前端/后端和强大的 MoE 支持。

Shadow traffic - 影子流量：将真实生产请求镜像到候选部署。

SNAC (audio decoder) - SNAC（音频解码器）：一种高性能音频解码路径，通常与 TTS token 流配对使用。

Softmax：在注意力中将分数转换为概率，并在解码中将 logits 归一化为概率分布。

Sparsity (FLOPS) - 稀疏性（FLOPS）：在具有 2:4 结构化稀疏性的张量中，其中 50% 的值为 0，Tensor Core 可以跳过乘以 0 的操作。大多数推理是稠密的，而非稀疏的。

Special Function Unit (SFU) - 特殊功能单元：一种专用硬件单元，加速特定数学操作如正弦和余弦，将专门操作从 CUDA Core 上卸载。

Speculative decoding - 推测解码：一系列生成和验证 draft token 的策略，以在 decode 期间每次前向传播生成多个 token。

Streaming Multiprocessor (SM) - 流式多处理器：包含核心和缓存的 GPU 计算单元。

Structured output - 结构化输出：遵循特定 schema 的 LLM 输出。通过 logit biasing 将生成约束到提供的 schema 来创建，而非通过提示。

SXM (GPU form factor) - SXM（GPU 外形规格）：一种插槽式 GPU 模块，支持更高带宽连接并提供比 PCIe 更大的功率。SXM 外形规格的 GPU 通常具有更高的基础规格，是推理的标准。

Temperature - 温度：控制 token 选择中的随机性：较低值（如 0.1）使输出更具确定性；较高值（如 1.5）增加多样性。

Tensor Parallelism (TP) - 张量并行：将张量操作在节点内的 GPU 之间分拆。每个用户的延迟最佳；需要频繁的 all‑reduce 同步。

TensorRT：NVIDIA 的优化运行时，具有融合 kernel、量化和其他优化功能，用于高性能推理。

TensorRT-LLM：由 NVIDIA 构建的推理引擎，提供 Python API 以及 TensorRT 引擎和 PyTorch 后端执行路径，具有融合 kernel、量化和推测解码功能。

Thread - 线程：GPU 上的最小执行单元。Kernel 启动许多线程以实现大规模并行。

Throughput - 吞吐量：单位时间内的总工作量（如每秒总 token 数）。

Time to first byte (TTFB) - 首字节时间：返回输出第一个字节的时间，一种延迟指标。

Time to first token (TTFT) - 首 token 时间：返回输出第一个 token 的时间，一种延迟指标。

Token - 词元：LLM 中文本处理的基本单位。一个 token 是一个整数，代表一个字符串。在英语中，大多数 tokenizer 的 token:word 比率约为 4:3。

Tokenizer - 分词器：确定性地将字符串转换为 token 序列，反之亦然。不同模型有不同的 tokenizer，更高效的 tokenizer 可改善端到端延迟。

Tokens per second (TPS) - 每秒 token 数：参见 perceived TPS。一种衡量流式传输给最终用户的每秒 token 数量的延迟指标。

Training - 训练：使用反向传播和优化从数据中学习模型权重的过程。训练是计算密集型的，通常在大型 GPU 集群上运行，产生推理中使用的权重。

Transformer：生成式 AI 模型背后的基础架构。

Transformers (library) - Transformers 库：LLM 和其他基于 transformer 的模型的参考实现。

Triton Inference Server：NVIDIA 的生产服务框架，支持多种后端。

VAE (variational autoencoder) - 变分自编码器：在推理中用于从 latent space 解码到 pixel space，用于图像和视频生成模型（也用于训练期间从 pixel space 编码到 latent space）。

Vector database - 向量数据库：用于存储和查询嵌入模型创建的语义向量的数据库。

Vector similarity - 向量相似性：基于如余弦相似度等公式检查两个向量之间的接近程度。高相似度的向量编码相似的语义含义。

Vera CPU：基于 ARM 的 NVIDIA CPU，具有 CPU 和 GPU 之间的高带宽芯片间互连。在 Rubin GPU 架构代际中接替 Grace CPU。

Vision‑language model (VLM) - 视觉语言模型：接受图像、视频以及文本提示并输出文本的模型。

vLLM：一种广泛采用的推理引擎，具有广泛的模型和硬件支持以及良好的默认配置。

Vocabulary - 词汇表：LLM 用于表示数据的 token 总集合，通常超过 100,000 个。

Voice activity detection (VAD) - 语音活动检测：一种轻量级模型，将流/文件分割为包含语音的块，用于 ASR。

VRAM (device memory) - 显存（设备内存）：GPU 上的内存，用于权重、KV 和激活。总 VRAM 决定模型大小和 KV 余量；带宽决定 decode TPS。

WebSocket：轻量级双向流式传输协议。适用于非结构化音频块和实时用户体验。

Weights‑only quantization - 仅权重量化：降低线性层中模型权重的精度，同时保持 KV cache 和注意力等其他模型组件在更高精度。一种保守的量化方法，具有最佳的质量保留但最低的性能提升。

Workload plane (multi‑cloud) - 工作平面（多云）：具有计算资源的独立集群，运行推理并处理请求。
